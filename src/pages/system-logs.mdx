# System Logs

A terminal-style component for displaying real-time system logs, process outputs, or audit trails with auto-scrolling and status indicators.

## Import

```tsx
import { SystemLogs, type LogEntry } from '@/components/SystemLogs/SystemLogs'
```

## Installation

Add the system logs to your project using the Dashkit CLI:

```bash
bunx carllosnc/dashkit add systemlogs
```

## Usage

The `SystemLogs` component takes an array of `logs` and renders them in a dark, monospace container that mimics a terminal environment.

<Preview>
  <SystemLogs
    title="Cloud Deployment"
    session="worker-nodes-01"
    status="Running"
    statusColor="success"
    maxHeight={300}

```tsx
import { SystemLogs } from '@/components/SystemLogs/SystemLogs'

const logs = [
  { type: 'info', message: 'Starting deployment of version 2.4.0...', timestamp: '10:00:01' },
  { type: 'ok', message: 'Docker image pulled: alpine-node:20', timestamp: '10:00:05' },
  { type: 'info', message: 'Allocating resources in us-west-2...', timestamp: '10:00:08' },
  { type: 'stable', message: 'Health checks passed. Traffic routing enabled.', timestamp: '10:00:15' },
]

export default () => (
  <SystemLogs 
    title="Cloud Deployment"
    session="worker-nodes-01"
    status="Running"
    statusColor="success"
    maxHeight={300}
    logs={logs}
  />
)
```

## Log Levels

The component supports five distinct log levels, each with its own visual style:

- `ok`: Success messages (indicated by `[OK]`)
- `info`: Informational messages (indicated by `[INF]`)
- `warn`: Warning messages (indicated by `[WRN]`)
- `error`: Error messages (indicated by `[ERR]`)
- `stable`: Stability or state updates (indicated by `[STB]`)

## Customization

### Auto-scrolling

By default, the component automatically scrolls to the bottom when new logs are added. You can disable this behavior by setting `autoScroll={false}`.

### Header & Footer

- `title`: The label displayed in the terminal header.
- `session`: An optional session identifier displayed in the footer.
- `status`: A status label displayed as a badge in the footer.
- `statusColor`: The color of the status badge.
