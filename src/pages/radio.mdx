# Radio

A premium radio button component for single choices from a set, with support for labels and descriptions.

## Import

```tsx
import { Radio } from '@/components/Radio/Radio'
```

## CLI

Add the radio button to your project using the Dashkit CLI:

```bash
bunx dashkit-ui add radio
```

## Basic Usage

The basic radio button with a label. Usually part of a group sharing the same `name`.

<Preview>
  <div className="flex flex-col gap-4">
    <Radio label="Option 1" name="radio-demo" value="1" defaultChecked />
    <Radio label="Option 2" name="radio-demo" value="2" />
  </div>
</Preview>

```tsx
<div className="flex flex-col gap-4">
  <Radio label="Option 1" name="radio-group" value="1" defaultChecked />
  <Radio label="Option 2" name="radio-group" value="2" />
</div>
```

## Controlled

Works seamlessly with React's state management.

<Preview>
  <RadioDemo />
</Preview>

```tsx
import { useState } from 'react'

export function LanguageSelection() {
  const [value, setValue] = useState('react')

  return (
    <div className="flex flex-col gap-4">
      <Radio 
        label="React" 
        checked={value === 'react'} 
        onChange={() => setValue('react')} 
      />
      <Radio 
        label="Vue" 
        checked={value === 'vue'} 
        onChange={() => setValue('vue')} 
      />
    </div>
  )
}
```

## Props

Extends all native `RadioHTMLAttributes<HTMLInputElement>` (except `type`), plus these specialties:

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `label` | `string` | — | Label text rendered to the right. |
| `description` | `string` | — | Subtle description text rendered below the label. |
| `className` | `string` | — | Additional CSS classes for custom styling. |
| `ref` | `Ref<HTMLInputElement>` | — | Forwarded ref to the underlying radio input. |
