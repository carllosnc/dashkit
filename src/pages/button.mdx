# Button

A highly customizable button component supporting multiple variants, icons, and loading states.

## Import

```tsx
import { Button } from '@/components/Button/Button'
```

## Installation

Add the button to your project using the Dashkit CLI:

```bash
bunx carllosnc/dashkit add button
```

## Variants

Use the `variant` prop to switch between visual styles.

<Preview className="flex-wrap items-start">
  <Button variant="filled">Filled</Button>
  <Button variant="outlined">Outlined</Button>
  <Button variant="soft">Soft</Button>
</Preview>

```tsx
<Button variant="filled">Filled</Button>
<Button variant="outlined">Outlined</Button>
<Button variant="soft">Soft</Button>
```

## With Icons

Use `leftIcon` and `rightIcon` to add icons before or after the label.

<Preview className="flex-wrap items-start">
  <Button variant="filled" leftIcon={<FiDownload />}>
    Download Data
  </Button>
  <Button variant="outlined" rightIcon={<FiArrowRight />}>
    Next Step
  </Button>
</Preview>

```tsx
import { FiDownload, FiArrowRight } from 'react-icons/fi'

<Button variant="filled" leftIcon={<FiDownload />}>
  Download Data
</Button>

<Button variant="outlined" rightIcon={<FiArrowRight />}>
  Next Step
</Button>
```

## Loading State

Set the `loading` prop to show a spinner and disable the button. When loading, the left icon is replaced by the spinner and the right icon is hidden.

<Preview className="flex-wrap items-start">
  <Button variant="filled" loading>
    Saving Changes
  </Button>
  <Button variant="outlined" loading>
    Processing
  </Button>
</Preview>

```tsx
<Button variant="filled" loading>
  Saving Changes
</Button>
```

## Disabled

Use the native `disabled` prop. The button becomes non-interactive with reduced opacity.

<Preview className="flex-wrap items-start">
  <Button variant="filled" disabled>Filled Disabled</Button>
  <Button variant="outlined" disabled>Outlined Disabled</Button>
</Preview>

```tsx
<Button variant="filled" disabled>Filled Disabled</Button>
<Button variant="outlined" disabled>Outlined Disabled</Button>
```

## Icon Only

If you omit the `children` (label) and provide only an icon, the button will automatically switch to a circular/square layout with balanced padding. **Note:** Always provide an `aria-label` for accessibility when using icon-only buttons.

<Preview className="flex-wrap items-start">
  <Button variant="filled" leftIcon={<FiDownload />} aria-label="Download" />
  <Button variant="outlined" leftIcon={<FiArrowRight />} aria-label="Next" />
</Preview>

```tsx
<Button variant="filled" leftIcon={<FiDownload />} aria-label="Download" />
<Button variant="outlined" leftIcon={<FiArrowRight />} aria-label="Next" />
```

## Custom Styling

You can easily override or extend the default styles using the `className` prop. The component uses `tailwind-merge` internally, so your custom classes will automatically take precedence over the defaults without needing the `!` (important) modifier.

<Preview className="flex-wrap items-start">
  <Button className="rounded-xl bg-indigo-600 hover:bg-indigo-700">
    Custom Radius & Color
  </Button>
  <Button variant="outlined" className="border-red-500 text-red-500 hover:bg-red-50">
    Destructive Action
  </Button>
</Preview>

```tsx
// Custom classes automatically override defaults thanks to tailwind-merge
<Button className="rounded-xl bg-indigo-600 hover:bg-indigo-700">
  Custom Radius & Color
</Button>

<Button variant="outlined" className="border-red-500 text-red-500 hover:bg-red-50">
  Destructive Action
</Button>
```

## Props

The component extends all native `ButtonHTMLAttributes<HTMLButtonElement>` props (e.g. `onClick`, `type`, `disabled`, `className`), plus the following:

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `'filled' \| 'outlined' \| 'soft'` | `'filled'` | Visual style of the button. |
| `leftIcon` | `ReactNode` | — | Icon element rendered before the label. Replaced by spinner when `loading`. |
| `rightIcon` | `ReactNode` | — | Icon element rendered after the label. Hidden when `loading`. |
| `loading` | `boolean` | `false` | Shows a spinner, disables the button, and hides icons. |
| `disabled` | `boolean` | `false` | Disables the button with reduced opacity. |
| `className` | `string` | — | Additional CSS classes merged via `clsx`. |
| `ref` | `Ref<HTMLButtonElement>` | — | Forwarded ref to the underlying `<button>` element. |
