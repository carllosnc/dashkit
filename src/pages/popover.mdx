# Popover

A floating container that displays rich content anchored to a trigger element. Popovers are ideal for more complex content than tooltips, such as forms or descriptive text.

## Import

```tsx
import { Popover, PopoverTrigger, PopoverContent } from 'dashkit'
```

## Installation

Add the popover to your project using the Dashkit CLI:

```bash
bunx carllosnc/dashkit add popover
```

## Basic Usage

The `Popover` component uses a composition-based API. Wrap your trigger and content in the `Popover` provider.

```tsx
import { Popover, PopoverTrigger, PopoverContent, Button } from 'dashkit';

export default () => (
  <Popover>
    <PopoverTrigger asChild>
      <Button variant="outlined">Open Popover</Button>
    </PopoverTrigger>
    <PopoverContent align="center" side="bottom">
      <div className="p-4">
        <h4 className="font-bold">Settings</h4>
        <p className="text-sm text-muted-foreground">Manage your account preferences.</p>
      </div>
    </PopoverContent>
  </Popover>
);
```

## Trigger Mode

You can change how the popover is triggered using the `trigger` prop. It supports `click` (default) and `hover`.

<Preview>
  <div className="not-prose flex gap-8 justify-center p-8">
    <Popover trigger="click">
      <PopoverTrigger asChild>
        <Button variant="outlined">Click Me</Button>
      </PopoverTrigger>
      <PopoverContent side="bottom" align="center">
        <div className="text-xs">Triggered by click</div>
      </PopoverContent>
    </Popover>

    <Popover trigger="hover">
      <PopoverTrigger asChild>
        <Button variant="outlined">Hover Me</Button>
      </PopoverTrigger>
      <PopoverContent side="bottom" align="center">
        <div className="text-xs">Triggered by hover</div>
      </PopoverContent>
    </Popover>
  </div>
</Preview>

```tsx
<Popover trigger="click">...</Popover>
<Popover trigger="hover">...</Popover>
```

## Features

- **Automatic Collision Handling**: The popover will flip to the opposite side if it doesn't have enough space in the preferred direction.
- **Escape to Close**: Pressing the `Escape` key closes the popover immediately.
- **Click Outside**: Automatically closes when user clicks outside the content or trigger.
- **Dynamic Portaling**: Renders via `createPortal` to avoid z-index and overflow issues.
- **Framer Motion Animations**: Smooth scale and opacity transitions.
