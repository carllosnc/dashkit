# Scroll Area

A minimalist scroll area component that provides theme-aware, custom scrollbars across all browsers and operating systems.

## Import

```tsx
import { ScrollArea } from '@/components/ScrollArea/ScrollArea'
```

## Installation

Add the scroll area to your project using the Dashkit CLI:

```bash
bunx carllosnc/dashkit add scrollarea
```

## Usage

The `ScrollArea` component wraps any content and replaces native scrollbars with a minimalist, theme-aware design while maintaining native scroll performance and accessibility.

### Vertical Scrolling (Default)

The scroll area will automatically handle vertical overflow if the content height exceeds the container height.

<Preview>
  <VerticalExample />
</Preview>

```tsx
<div className="h-64 border rounded overflow-hidden">
  <ScrollArea>
    <div className="p-6 space-y-4">
      {/* Long content here */}
    </div>
  </ScrollArea>
</div>
```

### Horizontal Scrolling

Set the `orientation` prop to `horizontal` to enable horizontal scrolling only.

<Preview>
  <HorizontalExample />
</Preview>

```tsx
<ScrollArea orientation="horizontal" className="h-20">
  <div className="flex gap-4 p-4 w-[1000px]">
    {/* Wide content here */}
  </div>
</ScrollArea>
```

### Both Orientations

Use `orientation="both"` when you need independent scrolling in both directions.

```tsx
<ScrollArea orientation="both" className="h-64">
  <div className="w-[1000px] h-[1000px] p-8">
    {/* Large two-dimensional content area */}
  </div>
</ScrollArea>
```

## Features

- **Native Performance**: Uses standard `overflow: auto` for maximum smoothness and accessibility.
- **Theme-Aware**: Automatically switches scrollbar colors based on light/dark mode using project design tokens.
- **Custom Scrollbar Utility**: Leverages the project's `custom-scrollbar` CSS utility for a consistent 4px thin scrollbar.
- **Cross-Browser**: Standardizes appearance across modern browsers.
- **Low Dependency**: Zero external libraries; depends only on core project utilities.
