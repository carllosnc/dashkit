# Card

A premium container component for grouping content, imagery, and actions.

Dashkit's Card is designed as a **composed wrapper**, providing total layout control through granular sub-components.

## Installation

Add the Card component to your project using the CLI or via manual import.

### Installation

```bash
bunx carllosnc/dashkit add card
```

### Manual Import

```tsx
import {
  Card,
  CardHeader,
  CardTitle,
  CardDescription,
  CardContent,
  CardFooter
} from '@/components/Card';
```

## Usage Example

```tsx
import { 
  Card, 
  CardHeader, 
  CardTitle, 
  CardDescription, 
  CardContent, 
  CardFooter, 
  Button, 
  Badge 
} from 'dashkit-ui';

export default () => (
  <Card>
    <CardHeader extra={<Badge content="Live" color="info" pulse />}>
      <CardTitle>System Health</CardTitle>
      <CardDescription>Real-time infrastructure performance</CardDescription>
    </CardHeader>
    <CardContent>
       <div className="py-2">
         <p className="text-sm text-base-600 dark:text-base-400 font-medium">
           All services are operational. Last checked 2m ago.
         </p>
       </div>
    </CardContent>
    <CardFooter>
       <Button variant="outlined" className="w-full">View Global Status</Button>
    </CardFooter>
  </Card>
);
```

## API Reference

### Card (Root)
The main wrapper container.

| Prop | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| `bordered` | `boolean` | `true` | Whether to show a border around the card. |
| `shadowed` | `boolean` | `true` | Apply the modern layered shadow token. |
| `animate` | `boolean` | `false` | Whether to apply entrance animations. |
| `className` | `string` | — | Root container classes. |

### CardHeader
The top section, typically containing the title and description.

| Prop | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| `extra` | `ReactNode` | — | Content on the top right (e.g. badges, status icons). |
| `className` | `string` | — | Custom classes for the header container. |

### CardTitle & CardDescription
Supporting typography components for the header.

| Component | Tag | Description |
| :--- | :--- | :--- |
| `CardTitle` | `h3` | Large, bold title with tight tracking. |
| `CardDescription` | `p` | Smaller, medium-weight supporting text. |

### CardContent
The main body of the card with standard internal padding and flex growth.

| Prop | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| `className` | `string` | — | Custom classes for the content body. |

### CardFooter
The bottom action area with a subtle separator.

| Prop | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| `className` | `string` | — | Custom classes for the footer container. |
