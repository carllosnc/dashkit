# Navbar

A premium, responsive navigation header with support for branding, navigation, and action buttons using a flexible composition pattern.

## Import

```tsx
import { Navbar, NavbarBrand, NavbarLinks, NavbarActions } from '@/components/Navbar/Navbar'
```

## Installation

Add the navbar to your project using the Dashkit CLI:

```bash
bunx carllosnc/dashkit add navbar
```

## Basic Usage

The `Navbar` uses a composition pattern, allowing you to easily arrange branding, links, and actions.

<Preview>
  <Navbar sticky={false}>
    <NavbarBrand>
      <span className="text-lg font-bold tracking-tight">Dashkit</span>
    </NavbarBrand>

    <NavbarLinks>
      <a href="#" className="text-sm font-medium text-foreground">Docs</a>
      <a href="#" className="text-sm font-medium text-ds-500">Examples</a>
    </NavbarLinks>

    <NavbarActions>
      <Button variant="filled" size="sm">Explore</Button>
    </NavbarActions>
  </Navbar>
</Preview>

```tsx
<Navbar>
  <NavbarBrand>
    <span className="font-bold">Dashkit</span>
  </NavbarBrand>

  <NavbarLinks>
    <a href="/docs">Docs</a>
    <a href="/examples">Examples</a>
  </NavbarLinks>

  <NavbarActions>
    <Button size="sm">Explore</Button>
  </NavbarActions>
</Navbar>
```

## With Icons & Actions

You can combine text links with icon buttons and utilities in the actions area.

<Preview>
  <Navbar sticky={false}>
    <NavbarBrand>

        </button>

```tsx
import { FiGithub, FiBell } from 'react-icons/fi';

<Navbar>
  <NavbarBrand>Acme Corp</NavbarBrand>
  
  <NavbarActions>
     <IconButton icon={<FiBell />} variant="ghost" />
     <Divider orientation="vertical" className="h-6" />
     <IconButton icon={<FiGithub />} variant="ghost" />
  </NavbarActions>
</Navbar>
```

## Features

- **Responsive Design**: Links are automatically hidden on smaller screens while branding and actions remain.
- **Backdrop Blur**: Optional glassmorphism effect using the `blur` prop.
- **Sticky Behavior**: Automatically sticks to the top of its parent container or viewport.
- **Composition Based**: Use semantic sub-components for full layout control.
