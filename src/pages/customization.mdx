# Customization

Dashkit UI components are built to be deeply customizable through **CSS Design Tokens**. Instead of complex configuration files, Dashkit leverages **Tailwind CSS v4's CSS-first theme system** combined with a robust CSS variable architecture.

All global and component-specific styles are defined in `src/dashkit.css`.

## The Two-Layer Token System

Dashkit uses a specialized two-layer system for managing design tokens:

1.  **Tailwind Mappings (`@theme`)**: Maps semantic Tailwind names (like `bg-background` or `text-primary`) to underlying CSS variables.
2.  **Theme Definitions (`:root` & `.dark`)**: Defines the actual values (colors, spacing, radii) for those variables in both light and dark modes.

## 1. Core Variables

The core variables control the foundation of your application. You can find these in the `:root` and `.dark` blocks of `src/dashkit.css`.

### Base Colors
| Variable | Description |
| :--- | :--- |
| `--ds-bg` | Main application background |
| `--ds-fg` | Main text color (foreground) |
| `--ds-border` | Default border color |

### Surface Colors
| Variable | Description |
| :--- | :--- |
| `--ds-card` | Background for cards and containers |
| `--ds-card-fg` | Text color within cards |
| `--ds-popover` | Background for dropdowns and menus |

### Interactive Colors
| Variable | Description |
| :--- | :--- |
| `--ds-primary` | Primary brand color |
| `--ds-secondary` | Secondary action color |
| `--ds-accent` | Subtle highlight/selection color |

## 2. Form & Input Tokens

Input components use a specialized set of tokens to ensure consistency across the UI:

```css
:root {
  --ds-input-bg: var(--color-ds-100);
  --ds-input-border: var(--color-ds-300);
  --ds-input-focus: var(--color-ds-400);
  --ds-input-fg: var(--color-ds-900);
  --ds-input-ring: var(--color-ds-primary-100);
}
```

## 3. Global Radius

Dashkit uses a centralized radius system for a consistent look and feel:

| Variable | Default Value |
| :--- | :--- |
| `--radius-base` | `0.5rem` |
| `--radius-md` | `0.75rem` |
| `--radius-lg` | `1rem` |

Changing `--radius-base` will update almost all components in the library (Buttons, Inputs, Cards, etc.) as they rely on the `ds-rounded` utility.

## Customizing Your Theme

To customize Dashkit, simply edit the values in `src/dashkit.css`.

### Example: Customizing Roundness

If you want a more rounded, "pill" style for your application:

```css
@theme inline {
  --radius-base: 1rem;
  --radius-md: 1.5rem;
  --radius-lg: 2rem;
}
```

### Example: Custom Brand Color

To change the primary color to a custom purple:

```css
:root {
  --ds-primary: #7c3aed; /* Violet 600 */
  --ds-primary-fg: #ffffff;
}

.dark {
  --ds-primary: #a78bfa; /* Violet 400 */
}
```

## Creating Your Own Tokens

When building your own components, you can add new tokens to the `:root` block to keep your design consistent:

1.  **Define the token** in `dashkit.css`:
    ```css
    :root {
      --ds-special-bg: var(--color-ds-primary-500);
    }
    ```
2.  **Map it** in `@theme inline`:
    ```css
    @theme inline {
      --color-special-bg: var(--ds-special-bg);
    }
    ```
3.  **Use it** in your component:
    ```tsx
    <div className="bg-special-bg px-4 py-2">
      My custom themed component
    </div>
    ```
